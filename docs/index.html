Octra HFHE Challenge v2: Security Analysis

Independent audit of pvac_hfhe_cpp at commit 071b0e9, the exact code that produced secret.ct for octC5eR9pLGKbpzTbDgHowkFt8HW7LZYb2gzehzxHamxuAZ.

Summary

The v2 artifact does not appear to be recoverable from the public files alone. The single published break (v1, a different wallet) used an R_com commitment that acted as an offline guess checker. v2 removes that commitment from the wire and wraps every block with two independent masks, which removes the ability to verify a plaintext guess at all.

This page documents the scheme, the one public value that depends on the plaintext, and every attack path checked so far, including the LPN direction.

The scheme in one page

  • Field F_p, p = 2^127 - 1. A basis g^i for i in [0, B), B = 337, where g has order 337.
  • A value v is stored as a set of edges. Each edge is (layer_id, idx, sign, w, sigma). The weight w is a field element. The syndrome sigma = H*x xor e is only used by the proof system and is ignored during decryption.
  • Decryption: v = c0 + sum(sign * w * g^idx * R_inv[layer]).
  • R[layer] is a secret multiplicative mask, R = prf_R(sk, seed) = r1 * r2 * r3, each r_i a Toeplitz compression of an LPN syndrome under the secret key.
  • Text is split into 15 byte blocks (utils/text.hpp). Cipher 0 holds the byte length. Ciphers 1..k hold the blocks. secret.ct has 22 ciphers, so 1 length plus 21 blocks, up to 315 bytes.
  • v2 wrapping: each block is fuse(Enc(v+m), Enc(-m)) with a fresh uniform m and two independent base layers, so two independent masks R0 and R1.

The only public quantity that depends on the plaintext

Every edge weight has the form w = R * c. Summing a layer gives a value anyone can compute:

T_L = sum(sign * w * g^idx) = target_L * R_L

For a wrapped block that is T0 = (v + m) * R0 and T1 = (-m) * R1. The masks and m come from getrandom and are independent and uniform, so the pair (T0, T1) is uniform over the field regardless of v. This is information theoretic hiding, not just computational hardness.

Why the v1 attack is dead in v2

Candidate verification is impossible

For any guessed block value v, pick any m. Then R0 = T0 / (v + m) and R1 = -T1 / m are always consistent. Every candidate fits the public data equally well, so a BIP39 wordlist, a checksum, or a JSON shape cannot prune anything. There is no public function of a guess to test. v1 could test guesses only because R_com was a hash of all public inputs plus R, and that hash is not serialized in v2.

The wrapped pair does not add-leak

The earlier bounty2 leak needed the two addends to share mask structure. Here the two fused layers use independent masks. All 44 layers in secret.ct carry distinct seeds and 0 are native source layers, verified with the probe tool. So T0 + T1 never collapses into a function of v alone.

Empirical known plaintext test

A distinguisher encrypts many blocks of v = 0 and many of a known nonzero value, round trips them through the wire format, and compares every public statistic. Result at N = 300:

v=0    : meanU(T0)=0.4999 meanU(T1)=0.5038 meanU(T0+T1)=0.5204 avgEdges=47.3 zeroSum=0
v=known: meanU(T0)=0.4895 meanU(T1)=0.4754 meanU(T0+T1)=0.4715 avgEdges=47.4 zeroSum=0

All statistics agree within sampling error. Edge count depends only on depth. There is no public zero test. With full ground truth, no distinguisher separates the two classes.

The LPN direction

The mask is R = r1 * r2 * r3, each r_i a Toeplitz compression of an LPN syndrome y = A*s xor e with secret dimension k = 4096 and noise tau = 1/8.

No observable instance

An LPN solver needs the pairs (A_r, y_r). Both are secret here. The row generator A is an AES-CTR stream keyed by the secret sk.prf_k. The syndrome y is folded into R through a secret Toeplitz key and never serialized. R itself is never public, only T = v * R is. Exposed LPN equations: zero. The instance is used purely as a PRF, so no solver can even be constructed.

Infeasible even if exposed

Gauss / ISD (needs k noise-free rows)   about 2^789
BKW ballpark (k / log2 k)               about 2^341
best known asymptotic exponents         above 2^128 for k >= 512, tau = 1/8

Reproduce

g++ -std=c++17 -O2 -march=native \
    -Ipvac_hfhe_cpp/include -Isource tools/probe.cpp -o probe
./probe .                     # per layer T_L = target * R for all 22 ciphers
python3 tools/lpn_estimate.py # LPN hardness plus the zero data blocker
python3 tools/lpn_demo.py     # ISD works when A and y are exposed (toy k=20)
