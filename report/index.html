Octra HFHE Challenge v2 · independent security analysis

The v2 artifact hides its plaintext perfectly. Here is the proof.

A full audit of pvac_hfhe_cpp @ 071b0e9 — the exact code that produced secret.ct — against every attack that broke earlier versions. All of them are closed, three independent ways.

T=v·R
T — publicly computable per layer v — the plaintext (unknown) R — secret mask, LPN-hard

Octra's own docs state the whole security model in one line: the only public value is T = v·R, and R = prf_R(sk, seed) is LPN-hard to recover. Everything below is a demonstration that the v2 wire format leaks nothing beyond that single quantity — and that the quantity hides v completely.

Verdict. secret.ct for octC5eR9…HamxuAZ is not recoverable from the public files by any shortcut: no oracle, no statistical bias, no algebraic mask cancellation, no weak randomness, no pk→sk leak, and no observable LPN instance. v1 (the solved wallet) relied on a candidate-checking R_com oracle that v2 deletes.

Only one public quantity depends on the plaintext

Each block is decrypted as v = Σ sign·w·gidx·R⁻¹. Summing a layer's edges gives T_L = Σ sign·w·gidx = target_L·R_L, computable from public data. Through the authors' own "public linear invariants" lens: any other linear combination of the weights equals R·(Σλᵢcᵢ), whose value is uniformly random because the underlying coefficients are free. T_L is the only value-bearing public functional — and it is masked by an independent uniform R.

Why v2 defeats the v1 attack — twice

v2 wraps every block as two independent-mask layers: T₀ = (v+m)·R₀, T₁ = (−m)·R₁, with fresh uniform m.

Proof 1 · candidate verification is impossible

For any guessed block value v, pick any m; then R₀ = T₀/(v+m) and R₁ = −T₁/m always fit. Every plaintext is equally consistent with the public data, so no BIP39 wordlist, checksum, or format constraint can even test a guess. The R_com hash that made v1's offline check possible is not serialized in v2.

Proof 2 · perfect hiding

With R₀, R₁, m independent and uniform (all from getrandom), the pair (T₀, T₁) is uniform over (𝔽ₚ*)² independently of v. This is information-theoretic hiding, not merely computational — confirmed empirically: a known-plaintext distinguisher finds zero separation.

The LPN angle — attempted, and where it dies

The mask is R = r₁·r₂·r₃, each rᵢ a Toeplitz-compressed LPN syndrome y = A·s ⊕ e over a k=4096, τ=1/8 secret s. Two walls:

No observable instance. The row generator A is AES-CTR keyed by secret sk.prf_k; the syndrome y is folded into R and never serialized; R itself is never public (only v·R is). Exposed LPN equations: zero. The instance is a PRF — no solver can even be constructed. A toy ISD solver recovers a k=20 secret in ~40 tries only because there both A and y are public.

Infeasible even if exposed. Cost estimates for k=4096, τ=1/8:

Gauss / ISD (needs k noise-free rows)   ~2^789
BKW ballpark (k / log2 k)               ~2^341
best-known asymptotic exponents         > 2^128  (for k ≥ 512, τ = 1/8)
