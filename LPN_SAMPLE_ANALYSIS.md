# LPN Sample Analysis (July 11 Update)

The challenge creator (@lambda0xE) released LPN samples from `sk.bin` at commit `d9d29d5`:

> "challenge 2 update - now there are lpn samples from sk.bin here (something that would never happen irl unless someone leaked them publicly) you can use these materials in cryptanalysis"

## What was released

44 JSONL files in `lpn_samples/`, one per cipher×layer (22 ciphers × 2 layers):
- Header: metadata (n=4096, t=16384, tau=1/8, seed info, public_T)
- 16384 rows: each with `{i, y, a}` where:
  - `a`: 4096-bit binary vector (the LPN matrix row, as hex)
  - `y`: syndrome bit (0 or 1)
  - `y = <a, s> XOR e` where `s = sk.lpn_s_bits` (4096-bit secret) and `e` is Bernoulli(1/8) noise

## The task

> "The task is clear now: recover S from y = <A, S> xor e, with tau = 1/8 and n = 4096."

Recovering `s` would allow computing `R` for any seed, and then decrypting `secret.ct`.

## Attack attempts

### Gaussian elimination (single instance)
- 16384 equations in 4096 unknowns (4x overdetermined)
- Noise rate 1/8 means ~2048 noisy equations
- Result: 37% syndrome error rate — noise propagation corrupts the solution
- The pivots themselves can be noisy, cascading through elimination

### Multi-instance Gaussian elimination + majority voting
- 44 independent instances sharing the same `s`
- 720,896 total equations
- Each instance gives a noisy estimate of `s` via Gaussian elimination
- Majority voting across 44 estimates: ~50% error rate — each estimate is essentially random due to noise propagation in elimination

### Iterative bit-flipping
- Start from random `s`, flip bits that reduce total error count
- Correct `s` should give ~90K errors (12.5%), random gives ~360K (50%)
- Result: stuck at ~50% error — the signal from flipping individual bits is too weak in 4096-dimensional space

### SAT/z3 approach
- 4096 Boolean variables with 16384 XOR constraints
- Too many variables for z3 to handle directly

## What would work

The LPN problem with n=4096, tau=1/8 is believed hard under standard assumptions:
- Prange ISD: ~2^789
- BKW: ~2^341  
- Best asymptotic (Esser et al.): >2^128 for n≥512

However, having 44 independent instances is a significant advantage not present in standard LPN analysis. The effective overdetermination (176x) is very high.

Possible approaches:
1. **Specialized ISD**: Adapt Information Set Decoding for multiple instances
2. **Lattice reduction**: Convert to BDD with the 44 instances as additional constraints
3. **BKW with tables**: Use the extra instances to build larger tables
4. **AES key recovery**: The LPN matrix A is generated by AES-CTR keyed with SHA256(prf_k || known). Recovering the AES key from A would directly give prf_k.

## Verification tool

`source/tools/verify_lpn_sample_binding.cpp` validates that the LPN samples are bound to the correct `secret.ct` and `pk.bin`.
